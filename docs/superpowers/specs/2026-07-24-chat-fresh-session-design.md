# Neru 자동 발화 1회 및 채팅 새 세션 설계

## 목표

Neru가 사용자 응답 없이 자동 발화를 반복하지 않도록 연속 자동 발화 상한을 1회로 제한한다. 채팅 입력창의 쓰레기통 버튼은 기존 세션의 메시지만 비우지 않고, 현재 세션을 삭제한 뒤 새로운 세션 ID로 대화를 시작한다.

## 결정

- 자동 발화 스케줄러의 기본 `maxConsecutive`를 2에서 1로 변경한다.
- 실제 사용자 메시지가 들어오면 기존처럼 자동 발화 횟수를 초기화한다.
- 쓰레기통 버튼은 현재 활성 세션을 영구 삭제하고 같은 캐릭터의 새 세션을 활성화한다.
- 다른 과거 세션으로 자동 전환하지 않는다.
- 시스템 프롬프트와 Neru 페르소나는 변경하지 않는다.
- 기존 OAuth PR #39의 브랜치에서 구현하고 검증 후 같은 PR에 푸시한다.

## 검토한 대안

1. 현재 세션을 삭제하고 새 세션을 만드는 방식.
   - 쓰레기통 의미와 일치하고 이전 대화가 다음 모델 요청에 포함되지 않는다.
   - 사용자가 승인한 방식이다.
2. 현재 세션을 보존하고 새 세션으로 전환하는 방식.
   - 복구는 쉽지만 삭제 버튼의 의미와 맞지 않고 원치 않는 세션이 남는다.
3. 기존 세션 ID에서 메시지만 초기화하는 방식.
   - 현재 동작이며 새 세션이라는 요구를 만족하지 않는다.

## 상태 전이

쓰레기통 클릭 시 authority 렌더러가 다음 순서로 처리한다.

1. 기존 활성 세션 ID와 캐릭터 ID를 캡처한다.
2. 기존 세션의 대기 전송을 취소하고 채팅 컨텍스트와 스트림 표시를 초기화한다.
3. 같은 캐릭터에 새 세션을 생성하고 활성화한다.
4. 새 세션 생성이 성공한 뒤 기존 세션을 삭제한다.
5. chat-sync가 변경된 활성 세션과 메시지 스냅샷을 다른 창에 전파한다.

새 세션을 먼저 생성하므로 생성 실패 시 기존 세션을 잃지 않는다. 기존 세션 삭제가 실패하더라도 새 세션은 활성 상태로 유지하고 오류를 호출자에게 전달한다.

## 컴포넌트 변경

- `use-proactive-speech.ts`에서 자동 발화 기본 상한과 문서를 1회로 변경한다.
- 채팅 세션 저장소에 현재 세션을 새 세션으로 교체하는 단일 상태 전이 API를 둔다.
- 채팅 유지보수 저장소는 기존 세션의 생성·컨텍스트·스트림 정리를 담당한다.
- stage-tamagotchi chat-sync의 `cleanup` 명령을 새 세션 시작 의미로 교체하고 authority에서만 영속 상태를 변경한다.
- `InteractiveArea.vue`의 쓰레기통 핸들러는 새 세션 요청을 호출한다. 기존 삭제 분석 이벤트는 삭제된 메시지 수와 함께 유지한다.

## 검증 기준

1. 별도 `maxConsecutive` 설정 없이 자동 발화가 한 번 발생한 뒤 타이머가 다시 예약되지 않는다.
2. 사용자 입력을 기록하면 자동 발화가 다시 한 번 가능해진다.
3. 쓰레기통 요청 후 활성 세션 ID가 이전과 달라진다.
4. 새 세션에는 시스템 초기 메시지만 존재한다.
5. 기존 세션은 로컬 세션 목록과 저장소에서 제거된다.
6. 같은 캐릭터의 다른 과거 세션이 있어도 그 세션으로 전환하지 않는다.
7. 위성 렌더러에서 요청해도 authority가 한 번만 새 세션을 생성한다.
8. 시스템 프롬프트 파일에는 변경이 없다.

## 범위 밖

- Neru 시스템 프롬프트 및 페르소나 문구 수정.
- 자동 발화 주제 중복을 의미적으로 판별하는 유사도 검사.
- 세션 삭제 확인 대화상자 추가.
- 채팅 세션 목록 UI 변경.
