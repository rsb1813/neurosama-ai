# Codex 프로젝트 컨텍스트 초기화 설계

## 목표

새 Codex 작업이 저장소를 열었을 때 코드와 문서를 처음부터 다시 탐색하지 않아도 현재 구조, 실행 경로, 상태 기준, 검증 명령, 위험 요소를 빠르게 파악하게 한다. 동시에 사람용 문서의 오래된 상태 표시를 최근 Git 이력과 맞춘다.

## 확인된 문제

- 저장소 루트에 프로젝트용 `AGENTS.md`가 없어 Codex가 자동으로 읽을 짧은 진입점이 없다.
- `WORKSPACE.md`는 웹 검색을 다음 작업으로 기록하지만 웹 검색은 PR #26으로 이미 머지됐다.
- `ROADMAP.md`와 `README.md` 일부는 barge-in을 미완료로 표시하지만 PR #21로 머지됐다.
- 능동 발화는 로컬 `feat/neru-proactive-speech` 브랜치에서 구현과 자동 검증이 끝났으나 원격 브랜치와 PR이 없다.
- `checklist.md`와 `context-notes.md`는 세부 이력 보존에는 유용하지만 현재 상태를 빠르게 읽는 진입점으로는 너무 길고 과거 구조 설명이 섞여 있다.

## 접근법 비교

### 루트 `AGENTS.md`만 추가

Codex 자동 로딩은 해결되지만 기존 문서의 상태 불일치를 그대로 남긴다. 같은 사실이 `AGENTS.md`와 다른 문서에서 다르게 보일 가능성이 높다.

### 루트 `AGENTS.md` 추가와 상위 상태 문서 동기화

`AGENTS.md`는 안정적인 구조와 작업 규칙만 담고, 변동이 잦은 상태는 `WORKSPACE.md`에 둔다. `ROADMAP.md`와 `README.md`는 각자의 목적에 필요한 요약만 최신화한다. 자동 로딩과 사람용 문서의 일관성을 함께 해결하므로 이 방식을 선택한다.

### 기존 문서만 정리

중복 파일은 늘지 않지만 Codex가 저장소 진입 시 어떤 문서를 어떤 순서로 읽어야 하는지 자동으로 알 수 없다.

## 문서 역할

### `AGENTS.md`

Codex가 자동으로 읽는 짧고 안정적인 프로젝트 진입점이다. 다음 내용만 둔다.

- neru의 제품 목표와 고정된 언어 흐름.
- 실제 런타임 구성과 핵심 경로.
- 문서별 역할과 읽기 순서.
- 수정 영역별 최소 검증 명령.
- 머지 상태와 로컬 브랜치 상태를 혼동하지 않는 규칙.
- 저장소 특유의 안전 불변식과 PR 리뷰 지침.
- `airi/` 아래에서는 더 가까운 `airi/AGENTS.md`도 적용된다는 안내.

세부 진행률, 긴 구현 이력, 일회성 브랜치 목록은 넣지 않는다.

### `WORKSPACE.md`

현재 시점의 단일 상태판으로 유지한다. 머지 완료, 로컬 진행 중, 알려진 문제, 즉시 다음 작업을 분리한다. 능동 발화는 “구현·자동 검증 완료, 수동 런타임 검증과 푸시·PR 대기”로 기록한다.

### `ROADMAP.md`

제품 비전과 하위 프로젝트 상태만 관리한다. barge-in, 장기 기억, 웹 검색의 머지 상태와 능동 발화의 로컬 진행 상태를 반영하되 구현 세부 로그는 넣지 않는다.

### `README.md`

처음 방문한 사람을 위한 제품 소개와 실행 방법을 유지한다. MVP 체크박스 중 현재 사실과 다른 항목만 고친다.

### `checklist.md`와 `context-notes.md`

과거 작업의 세부 체크와 결정 기록으로 보존한다. 이번 작업 내역을 짧게 추가하되 전체를 현재형 문서로 재작성하지 않는다.

## 상태 표현 규칙

- `master` 또는 GitHub PR에 머지된 사실과 로컬 브랜치 작업을 별도 항목으로 표시한다.
- 자동 테스트 통과와 수동 런타임 검증을 구분한다.
- 커밋, PR, 브랜치처럼 검증 가능한 근거를 상태 설명 가까이에 둔다.
- 변동이 잦은 “다음 작업”은 `WORKSPACE.md`에만 상세히 적고 다른 문서는 링크한다.
- 문서가 충돌하면 Git과 실제 코드가 우선이며, 확인 후 관련 상위 문서를 같은 변경에서 동기화한다.

## PR 리뷰 지침

루트 `AGENTS.md`에 `Review guidelines`를 둬 로컬 Codex 리뷰와 GitHub Codex 리뷰가 같은 기준을 사용하게 한다.

- 인증, 로컬 서비스 경계, 자격 증명 노출, 경로 입력, IPC 입력 검증을 우선 확인한다.
- `English <ko>Korean</ko>` 출력 불변식과 영어 전용 TTS 경로를 깨는 변경을 높은 위험으로 본다.
- 능동 발화와 barge-in의 중복 턴, 히스토리 오염, 취소·정리 누락을 확인한다.
- 버그 수정에는 재현 테스트가 있는지 확인한다.
- 기존 AIRI 상류 코드의 광범위한 스타일 문제는 이번 변경과 직접 관련 없으면 리뷰 결함으로 올리지 않는다.

## 검증

- 문서에 남은 오래된 상태 문구를 검색한다.
- `AGENTS.md`의 모든 경로와 명령이 실제로 존재하는지 확인한다.
- Git 로그와 브랜치 상태가 `WORKSPACE.md`의 머지·로컬 구분과 일치하는지 대조한다.
- Markdown 링크의 상대 경로가 유효한지 검사한다.
- 문서 변경이므로 제품 빌드 전체 대신 문서 정적 검사와 Git diff 검토를 수행한다.

## 완료 조건

- 루트 `AGENTS.md`만 읽어도 프로젝트 구조, 문서 읽기 순서, 검증 방법, 핵심 불변식을 파악할 수 있다.
- `WORKSPACE.md`, `ROADMAP.md`, `README.md`가 웹 검색, barge-in, 능동 발화 상태를 서로 모순 없이 표현한다.
- 로컬 능동 발화 브랜치를 `master`에 머지된 기능으로 잘못 표시하지 않는다.
- 상세 과거 기록을 불필요하게 재작성하지 않는다.
